Rstat.us Code Walkthrough
¶

Welcome to rstat.us' code!

Rstat.us uses Ruby on Rails 3.1. It follows the structure of a standard Rails app, so if you have not used Rails before, you may want to take a look at the Getting Started Rails Guide. Or feel free to dive in to rstat.us!

You can skip to the documentation for any file by using the "Jump to..." menu in the upper right of any documentation page.

This page is an index of the files in rstat.us' repository with a quick explanation of what each file is responsible for.
¶

Most of the code for the logic of rstat.us lives in the app directory.

Models are the business objects of the system.

 
  |- models/
¶

An Author is an entity that produces Updates. Authors may have corresponding rstat.us Users as well, but they may also be users on another OStatus site.

 
  |-- author.rb
¶

An Authorization is an ability of a User to log in to rstat.us. Currently, the only ways an Authorization is created are if a User logs in to rstat.us with Twitter or a User associates their Twitter account with their existing rstat.us email account.

 
  |-- authorization.rb
¶

Feeds are representations of PuSH enabled Atom feeds. Each rstat.us User has a Feed of their updates, and rstat.us keeps a local Feed for each remote Feed that an rstat.us User subscribes to.

 
  |-- feed.rb
¶

Notifier is a class that sends emails to Users such as password reset links or email confirmation links.

 
  |-- notifier.rb
¶

An Update is an individual status posted either by an rstat.us User or a remote user being followed by someone on rstat.us.

 
  |-- update.rb
¶

A User is someone who has signed in to rstat.us. All Users have Authors and Feeds associated with them.

 
  |-- user.rb
